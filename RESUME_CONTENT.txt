ESG Portfolio Optimizer
Machine Learning for Sustainable Investing
GitHub: https://github.com/yourusername/esg-portfolio-optimizer

Built an end-to-end quantitative investment system that scores S&P 500 
companies on Environmental, Social, and Governance (ESG) criteria and 
constructs optimized portfolios using machine learning.

Key Achievements:
• Achieved 11.29% CAGR with 0.743 Sharpe ratio, outperforming S&P 500 by 
  2.49% annually over 2022-2024 backtest period
• Demonstrated ESG quality as a predictive factor, with better ESG periods 
  generating +31.76% premium over worse ESG periods
• Trained XGBoost gradient boosting models on 462 companies with Sustainalytics 
  ESG ratings (91.8% coverage)
• Constructed sector-neutral portfolio of ~100 stocks with quarterly 
  rebalancing and risk management

Technical Implementation:
• Data Engineering: Built automated ETL pipeline integrating 7 data sources 
  (SEC EDGAR, GDELT, EPA, Yahoo Finance, Kaggle) with ~477 MB raw data
• Feature Engineering: Extracted 16 ESG features using NLP on SEC 10-K filings, 
  news sentiment analysis, emissions data, and financial metrics
• Machine Learning: Trained 3 XGBoost models (Environment, Social, Governance) 
  with MLflow experiment tracking and hyperparameter optimization
• Portfolio Construction: Implemented sector-neutral selection (top 20% ESG 
  scorers per sector) with quarterly rebalancing and transaction costs
• Production Architecture: Deployed FastAPI REST API, Streamlit interactive 
  dashboard, Docker containers, and GitHub Actions CI/CD pipeline

Technologies: Python, XGBoost, Pandas, NumPy, Scikit-learn, MLflow, FastAPI, 
Streamlit, Docker, GitHub Actions, Parquet

                    INTERVIEW TALKING POINTS

ELEVATOR PITCH (30 seconds):
"I built an ESG portfolio optimizer that uses machine learning to score 
companies and construct portfolios. It integrates data from SEC filings, news, 
and emissions databases, trains XGBoost models on industry-standard ESG 
ratings, and achieves 11.29% annual returns with a 0.743 Sharpe ratio, beating 
the S&P 500 by 2.49% per year."

TECHNICAL DEPTH (2 minutes):
"The system has a multi-stage pipeline. First, I ingest data from 7 sources - 
SEC EDGAR for corporate disclosures, GDELT for news sentiment, EPA for 
emissions, Yahoo Finance for prices and fundamentals, and Kaggle for training 
labels. 

I engineer 16 features using NLP on SEC filings - counting climate mentions, 
labor keywords, governance terms - plus emissions intensity and financial 
metrics. These features are normalized within sectors for fair comparison.

I train 3 XGBoost models - one each for Environment, Social, and Governance - 
on 462 companies with Sustainalytics ratings. The models achieve moderate 
correlation with expert consensus, but more importantly, they're predictive of 
returns.

For portfolio construction, I select the top 20% ESG scorers within each 
sector, weight them equally within sectors for sector-neutrality, and rebalance 
quarterly. The backtest shows 11.29% CAGR with 0.743 Sharpe ratio.

The production architecture includes MLflow for experiment tracking, FastAPI 
for the REST API, Streamlit for the dashboard, and Docker for deployment. 
Everything is configuration-driven and has CI/CD via GitHub Actions."

BUSINESS IMPACT:
"The key insight is that ESG quality is a real factor. When the portfolio held 
companies with better ESG scores, it returned +27.87% annualized. When it held 
worse ESG companies, it lost -3.89%. That's a 31.76% premium from ESG quality.

This demonstrates that ESG isn't just about values - it's about identifying 
well-managed, lower-risk companies that outperform. The 2.49% annual alpha over 
S&P 500 shows this approach adds real value."

CHALLENGES OVERCOME:
"The biggest challenge was data integration. SEC filings are unstructured text, 
GDELT has rate limits, EPA data needs fuzzy matching to tickers. I built 
resume capability for long-running ingestion, implemented proper rate limiting, 
and created a robust ETL pipeline.

Another challenge was avoiding look-ahead bias in backtesting. I ensured the 
backtest only uses data available at each point in time, includes transaction 
costs, and tests on out-of-sample data (2022-2024) that wasn't used for 
training."

WHAT I LEARNED:
"I learned a lot about production ML systems - the importance of configuration 
management, experiment tracking, and clean architecture. I also learned about 
quantitative finance - portfolio construction, risk metrics, and backtesting 
methodology. Most importantly, I learned that ESG is a measurable, predictive 
factor, not just a qualitative concept."

NEXT STEPS:
"Future enhancements include integrating advanced NLP models like FinBERT for 
sentiment analysis, adding real-time data feeds for live portfolio management, 
and expanding to international markets. I'd also like to implement mean-
variance optimization for more sophisticated portfolio weighting."
